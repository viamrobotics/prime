# @viamrobotics/prime-ui

Viam's Svelte 5 component library — the v2 of prime. Components live in `src/lib/`; `src/routes/` is a showcase / dev sandbox for working on them locally, not part of the published artifact.

## Layout

- `src/lib/` — published source. Add components here. `src/lib/index.ts` re-exports the public API.
- `src/routes/` — SvelteKit playground app. Used as the dev sandbox locally; built statically and embedded under [apps/docs](../../apps/docs/) at `/playground/prime-ui/` for the deployed docs site. Not published to npm.
- `dist/` — generated by `svelte-package` during `pnpm build`; what npm sees via the `files` field.
- `build/` — generated by `vite build` (static SvelteKit output). Gitignored; consumed by [apps/docs/scripts/build-playgrounds.mjs](../../apps/docs/scripts/build-playgrounds.mjs).

## Stack

| Layer       | Tool                                                                      |
| ----------- | ------------------------------------------------------------------------- |
| Framework   | Svelte 5 (runes) via SvelteKit                                            |
| Adapter     | `@sveltejs/adapter-static` (playground deploys as static gh-pages bundle) |
| Bundler     | Vite                                                                      |
| Styles      | Tailwind CSS v4                                                           |
| Tokens      | `@viamrobotics/tailwind-config`                                           |
| Tests       | Vitest (browser mode for `.svelte`)                                       |
| Typecheck   | `svelte-check`                                                            |
| Lint/format | ESLint + Prettier                                                         |

## Workspace deps

`@viamrobotics/tailwind-config` is consumed via `workspace:^`. It's listed in `peerDependencies` (downstream consumers must install it for tokens to resolve) and also in `devDependencies` (so this package's own build/showcase can `@import '@viamrobotics/tailwind-config'`). At publish time pnpm rewrites `workspace:^` to a concrete `^X.Y.Z` range.

## Commands

Run from the repo root with a `--filter`, or from inside this package directly:

```bash
pnpm --filter @viamrobotics/prime-ui dev          # start showcase dev server
pnpm --filter @viamrobotics/prime-ui build        # vite build + svelte-package + publint
pnpm --filter @viamrobotics/prime-ui check        # svelte-check
pnpm --filter @viamrobotics/prime-ui test         # vitest, one shot
pnpm --filter @viamrobotics/prime-ui test:unit    # vitest, watch mode
pnpm --filter @viamrobotics/prime-ui lint         # prettier --check + eslint
pnpm --filter @viamrobotics/prime-ui format       # prettier --write
```

The root scripts (`pnpm build`, `pnpm check`, `pnpm lint`, `pnpm test`) fan these out across the workspace.

## Static build / playground deployment

`svelte.config.js` reads `BASE_PATH` from the environment so the same build works at multiple deploy paths. Locally and on npm: empty (`/`). On GitHub Pages main deploy: `/prime/playground/prime-ui`. On PR previews: `/prime/pr-preview/pr-<N>/playground/prime-ui`. The docs site sets this for you via [apps/docs/scripts/build-playgrounds.mjs](../../apps/docs/scripts/build-playgrounds.mjs); no manual env wrangling needed for normal development.

All routes must be prerenderable — `src/routes/+layout.ts` enforces this with `export const prerender = true`. If you add a route that can't prerender, the static build will fail; either make it prerenderable or rethink whether it belongs in the playground.

## Conventions

- Svelte 5 runes only — no Svelte 4 syntax. See [.claude/rules/svelte.md](../../.claude/rules/svelte.md).
- Components use Tailwind utility classes that reference tokens declared in `@viamrobotics/tailwind-config` (e.g. `bg-power-button`, `text-heading`). Never hard-code colors or spacing — use the design tokens.
- New design tokens belong in `@viamrobotics/tailwind-config`'s `tailwind-config.css`, not in components.
- Token values and component patterns must match the Viam design system at https://design.viam.com/. See [.claude/rules/viam-context.md](../../.claude/rules/viam-context.md).
